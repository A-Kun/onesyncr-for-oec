Did you follow your plan(s) exactly (huh!), or did you have to re-plan? Why and when?

Before we started coding, we analyzed all of the requirements given by the client and we wrote personas and user stories. During the first week of the project, we achieved the basic functionalities for checking the required databases and getting data from them. We wrote a crawler to crawl data and parse crawled data into dictionaries. We implemented an email sending function which connects to the email server, sends out emails and generates HTML email contents. We also worked on GitHub API to create a pull request to GitHub and pass the URL of the pull request to the email module. Throughout this sprint, we realized that some features are just nice to have, so we lowered the priority of comparing and showing differences between databases. For the next release, since we basically finished the backend, we re-planed to add a user interface into the release. We developed the user interface and also fixed some technical issues like the compare function. Starting from this deliverable, we focused on implementing merge without any bugs and making our front end API work efficiently. We made sure basic features like parsing data and merging from differently formatted data is working completely fine by doing code inspection. Then from there we developed more features like allowing the client to edit the file and also improved the user interface.

What was your estimated project velocity? How did it change with each deliverable?

Overall, our project had a smooth transaction from deliverable 2 to deliverable 3. We put our user stories from deliverable 2 into our backlog and had an estimated project velocity of 60 story points (1 story point == 1 dev hour). We followed our plan and finished 39 story points in this sprint and moved the one user story that was not finished into the next sprint. For deliverable 4, the estimated velocity was 56 story points and the actual velocity was 36 story points. We moved two stories to the next. For the last deliverable, we finished all of the planned story points and discarded the lowest priority ones. The estimated velocity was 58 story points, and actual velocity was 51 points.

How does the work done for deliverable 5 compare to the work on all previous deliverables, both in terms of progress and end result?

As mentioned above, compared to all previous deliverables, deliverable 5 was more about code inspection (making sure everything works perfectly) and adding small features. We made good progress by quickly identifying and fixing issues in our project. It resulted in an end product that satisfies the basic requirements and works perfectly. We ended up discarding or simplifying some features that are non-essential, due to time constraints.

Submit an updated description of the system componets: what they are, what their responsibilities are, and what are the interactions betwwen them.

The system use browser server structure, there are front end view which directly interact with user. Whenever a user sign in or click on the button, a request will send to server side. Then server take the request and operate business logic and render results to front end page view. The system binding all merge algorithm (A module named main.py) with a HTTP request within server side. All HTTP handler inside service.py. When look for a update, the system will download files from all databases into local file folder first, then merge.py will compare files to files within a different folders. Once the files are merged, ready to make push API call to Github.

Describe and explain any changes you made.

We thought about use third party Non-relational database to improve performce, However, after gave a try and we found it is too complicated to implement in such a way. First we need to compare data in dictionary, which make the comparation much harder. Secondly, we need to find a way to convert dictionary to xml and xml to dictionary. Therefore, we come back to original designed and use multithreading module to improve better performance.
